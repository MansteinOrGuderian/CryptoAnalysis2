Мета комп'ютерного практикуму: Практична реалізація алгебраїчної атаки на фільтрувальний генератор гами; набуття
навичок роботи з системами комп’ютерної алгебри в SAGE.

\noindent Постановка задачі:
\begin{enumerate}
    \item Знайти функції мінімального степеня ідеалів $\langle f \rangle$ та $\langle f \oplus 1\rangle$ за допомогою побудови
        базису Грьобнера. Якщо побудова базису для одного з ідеалів $\langle f \rangle$ або $\langle f \oplus 1\rangle$ є занадто
        трудомісткою з точки зору обчислювальних ресурсів, то будувати лише один базис -- за умови, що цього буде достатньо для
        проведення атаки.
    \item Визначити кількість рівнянь, необхідних для відновлення початкового стану (якщо не вдається визначити аналітично,
        то емпірични шляхом підбирати). Побудувати систему рівнянь меншого степеня відносно початкового стану генератора.
    \item Знайти розв’язки отриманої системи рівнянь. Зауважимо, що початковий стан за умовою комп’ютерного практикуму
        є ненульовим вектором.
    \item Перевірити, що початковий стан відновлено правильно, шляхом генерування відрізка гами відповідної довжини й
        порівнявши його з вхідними даними.
\end{enumerate}

\subsection{Побудова базисів Грьобнера}

Кільце булевих поліномів $B_{64} = \mathbb{F}_2[x_0, \ldots, x_{63}]/\langle x_i^2 \oplus x_i \rangle$
ініціалізовано за допомогою мономіального впорядкування $degrevlex$ (степенево-зворотне лексикографічне) --
найефективніше для базисів Грьобнера (звичайний $lex$ був би корисним для виключення змінних, і їв при цьому більше
часу), яке є степеневим, що гарантує коректність застосування теореми Арса--Фожере (вона обґрунтовує застосування
базисів Грьобнера для пошуку функцій найменшого степеня в ідеалі).

\paragraph{Базис ідеалу $\langle f \rangle$.}
Час побудови: $\approx 578$~с. Потужність базису: 10047.

Функції мінімального степеня ідеалу $\langle f \rangle$: степінь~$d_1 = 2$, кількість: $1$.

Єдина функція мінімального степеня:
\begin{equation*}
    g_1 = x_{61} \cdot x_{21} \oplus x_{61}.
\end{equation*}
Зауважимо, що $g_1 = x_{61}(x_{21} \oplus 1)$, тобто $g_1 \in \langle f \rangle = \mathrm{Ann}(f \oplus 1)$
і $g_1 \cdot (f \oplus 1) = 0$.

\paragraph{Базис ідеалу $\langle f \oplus 1 \rangle$.}

Не будувався, точніше як... Побудова $\langle f \oplus 1 \rangle$ впала через нестачу пам'яті (вбудований sage алгоритм 
PolyBoRi (Polynomials over Boolean Rings) завершився з помилкою \texttt{"Built-in matrix-size exceeded!"} через 
перевищення внутрішнього ліміту на розмір матриці) Це саме та ситуація, яка описувалася в завданні: "будувати лише 
один базис, за умови, що цього буде достатньо для проведення атаки". Як раз наш випадок :)

\paragraph{Алгебраїчна імунність.}
Оскільки мінімальний степінь функцій в $\langle f \rangle$ дорівнює~$2$, а $\mathrm{AI}(f) = \min(d_1, d_2) \leq d_1$, 
маємо:
\begin{equation*}
    \mathrm{AI}(f) \leq 2.
\end{equation*}
Для порівняння: оптимальне значення для $n = 64$ (довжина регістру) становить $\lceil 64/2 \rceil = 32$. Таким 
чином, функція має надзвичайно низьку алгебраїчну імунність і є вразливою до алгебраїчної атаки~\cite{courtois2003}.

\subsection{Побудова системи рівнянь меншого степеня}

Через $e = 2$ позначимо степінь знайденої функції $g_1 \in \langle f \rangle$, що використовується для 
побудови рівнянь-наслідків. Кількість мономів степеня не більше~$e$ у $n = 64$ змінних обчислюється формулою:
\begin{equation*}
    M = \sum_{k=0}^{2} C_{64}^{k} = 1 + 64 + 2016 = 2081.
\end{equation*}

Оскільки побудувати базис $\langle f \oplus 1 \rangle$ не вдалось, то рівняння будуватимемо лише з тактів, де 
$\gamma_i = 0$. У цьому випадку $f(C^i \cdot \mathbf{x}) = 0$, і з огляду на $g_1 \in \langle f \rangle = \mathrm{Ann}(f \oplus 1)$ маємо
$g_1(C^i \cdot \mathbf{x}) = 0$.

P.S. Якби вдалось побудувати базис ідеалу $\langle f \oplus 1 \rangle$ і знайти в ньому функцію, що 
$g' \in \langle f \oplus 1 \rangle = \mathrm{Ann}(f)$, то аналогічно $g' \cdot f = 0$, і рівняння 
$g'(C^i \cdot \mathbf{x}) = 0$ можна було б записати для тактів, де $\gamma_i = 1$. У такому разі для побудови 
системи використовувались би \emph{усі} $N = 50000$ тактів. Однак і $25108$ нульових тактів значно перевищують 
необхідну кількість $M = 2081$, тому і одного ідеалу цілком достатньо.

Стан регістру в момент $i$ обчислюється символьно через рекурентне співвідношення ЛРЗЗ, що є значно ефективнішим 
в порівнянні з безпосереднім множенням супровідної матриці $C^i$. Усього побудовано \textbf{25108} нетривіальних 
рівнянь степеня~$2$. Час побудови системи: $\approx 17{,}4$~с.

\subsection{Розв'язання системи}

Систему розв'язуємо шляхом побудови базису Грьобнера (алгорит Бухбергера) -- об'єднаного ідеалу всіх рівнянь у 
кільці~$B_{64}$. Редукований базис Грьобнера має $63$ елементи виду $x_i \oplus x_{62}$ або просто мономи $x_i$, 
що означає наявність однієї вільної змінної (в нашому випадку $x_{62}$) і, відповідно, двох розв'язків: нульового 
та ненульового.

Час розв'язання: $\approx 25{,}4$~с. Обидва розв'язки знайдено за допомогою функції \texttt{variety()} ($\approx 1{,}1$~с).

\begin{remark}
    ~\par \noindent Як я для себе зрозумів -- базис Грьобнера це аналог методу Гауса для поліноміальних систем. Для 
    лінійних систем він буде еквівалентний Гаусу, для нелінійних -- узагальнюватиме його (оскільки Гаус не працює 
    з квадратичними рівняннями).
\end{remark}

\section{Висновки}

\begin{enumerate}
    \item Побудовали редукований базис Грьобнера для ідеалу $\langle f \rangle$ у кільці $B_{64}$ з мономіальним 
        впорядкуванням \texttt{degrevlex} (час $\approx 578$~с, потужність базису складає 10047). Побудова базису 
        $\langle f \oplus 1 \rangle$ виявилась неможливою через бібліотечне обмеження пам'яті, однак базис 
        $\langle f \rangle$ виявився цілком достатнім для атаки.
    \item Визначили верхню межу алгебраїчної імунності: $\mathrm{AI}(f) \leq 2$, що є надзвичайно низьким значенням 
        порівняно з оптимальним $\lceil 64/2 \rceil = 32$.
    \item Побудовали систему з $25108$ квадратичних рівнянь і розв'язали її за допомогою базису Грьобнера. 
        Вона має два розв'язки: нульовий (відкидається за умовою) та ненульовий $\mathbf{x} = \texttt{0x67a627812bd4d65e}$.
    \item Початковий стан перевірено шляхом генерування гами: всі $N = 50000$ бітів збіглись з вхідними даними.
\end{enumerate}

Отже, алгебраїчна атака є ефективним методом відновлення початкового стану фільтрувального генератора, що має 
низьку алгебраїчну імунність. Загальний час атаки (без врахування побудови базису Грьобнера) становить менше~30~с, 
що свідчить про \underline{повну криптографічну нестійкість} заданої фільтрувальної функції.
